Данное приложение создано в рамках тестового задания.
Суть приложения в том, чтобы работник компании мог создавать запрос для подготовки справок в бухгалтерии, а также отслеживать их статус. Бухгалтер должен видеть все заявки и изменять их статусы, в зависимости от того на каком этапе работы они находятся.

Для запуска приложения необходимо скачать проект, поместить файлы "Certificate.xml" и "Users.xml" в папку "Debug" по пути "ROSA\bin\Debug". После чего запустить файл "ROSA.sln".

Когда приложение запускается, то открывается форма для Работника. Для просмотра своих заявок пользователь может ввести свое имя в поле "Имя работника" и посмотреть свои заявки, нажав на кнопку "Смотреть запросы" (если заявок нет, то высветится соответствующее сообщение).
Для создания новой заявки пользователю необходимо заполнить все поля: "Имя работника", "Запрашиваемая справка", "Количество экземпляров", "Причина запроса". Элемент "Запрашиваемая справка" имеет в себе список справок: "2-НДФЛ", "Место работы", "Средний заработок" и "Другое". При выборе "Другое", активируется соответствующее текстовое поле, в которое можно ввести название другой справки. После заполнения полей необходимо нажать кнопку "Отправить". Если все поля заполнены корректно, то пользователь увидит все свои заявки, включая и новую в таблице. Если поля заполнены некорректно, то высветится предупреждение.

Для перехода в режим "Бухгалтер", необходимо нажать на соответствующую радиокнопку. Нажав на не появится форма для авторизации, чтобы войти необходимо ввести следующие данные: Имя пользователя - “Петров Петр Петрович”, Пароль “Бухгалтер”. Данное действие предусмотрено, чтобы только бухгалтер мог видеть все заявки и вносить изменения. В тестовом варианте существует только обид бухгалтер.  При успешной авторизации откроется таблица со всеми действующими запросами. Запросы записаны в порядке создания, т.е сверху старые, снизу новые. Бухгалтер может изменять "Статус" запроса, нажимая на соответствующую ячейку и выбирая нужный статус. Для сохранения изменений необходимо нажать на кнопку "Сохранить изменения".

Всего существует четыре статуса запроса:
"Создан" - присваивается при создании запроса пользователем;
"В работе" - подразумевает, что бухгалтер начал готовить справку;
"Готов" - справка готова и работник может ее забрать;
"Закрыт" - справка выдана работнику, после присвоения данного статуса, заявка удаляется из общего реестра.
